# EN/UK i18n Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Двуязычный сайт (en — дефолт, uk) на Payload CMS + Next.js App Router: локализованный контент в админке, роуты `/en/...` и `/uk/...` через next-intl.

**Architecture:** Payload core localization для контента (`localized: true` на полях, запросы с `locale`); все frontend-страницы переезжают в сегмент `app/(frontend)/[locale]/`; next-intl даёт middleware-редиректы, словари UI-строк и locale-aware навигацию. Слаги общие для обоих языков.

**Tech Stack:** Payload 3.86 (mongoose), Next.js App Router, next-intl v4, pnpm.

**Spec:** `docs/superpowers/specs/2026-07-17-i18n-en-uk-design.md`

## Global Constraints

- **НИКАКИХ `git commit`** — пользователь коммитит сам. Шаги «Commit» в этом плане отсутствуют намеренно; не добавлять их. `git mv` для переносов — можно (это staging, не commit).
- Пакетный менеджер — **pnpm**. Локали ровно две: `en` (дефолт, fallback) и `uk`; `localePrefix: 'always'`.
- Единый источник правды для списка локалей — `src/i18n/routing.ts`. В `payload.config.ts` список дублируется с комментарием-ссылкой (Payload-конфиг не должен импортировать из i18n).
- Код-стайл (обязателен, из правил пользователя): только `const`, никаких `let`/`for`/`while`; все колбэки в `map/filter/reduce/forEach` — именованные функции, инлайн-лямбды с логикой запрещены; guard clauses без optional chaining; не мутировать аргументы.
- После любого изменения Payload-полей: `pnpm generate:types` (и `pnpm generate:importmap`, если менялись admin-компоненты).
- Тестового раннера в репозитории нет — верификация каждой задачи: `npx tsc --noEmit` + запуск `pnpm dev` + curl/ручная проверка. Каждая задача заканчивается зелёным tsc (исключений нет).
- `next dev` может уже быть запущен пользователем — перед `pnpm dev` проверить `lsof -i :3000`.

- [ ] **Step 4: `src/utilities/getGlobals.ts`** (целиком)

```ts
import type { Config } from 'src/payload-types'

import configPromise from '@payload-config'
import { type DataFromGlobalSlug, getPayload, type TypedLocale } from 'payload'
import { unstable_cache } from 'next/cache'

type Global = keyof Config['globals']

async function getGlobal<T extends Global>(
  slug: T,
  depth: number,
  locale: TypedLocale,
): Promise<DataFromGlobalSlug<T>> {
  const payload = await getPayload({ config: configPromise })

  const global = await payload.findGlobal({
    slug,
    depth,
    locale,
  })

  return global
}

/**
 * Returns a unstable_cache function mapped with the cache tag for the slug.
 * Cache key includes the locale; the tag stays per-slug so one revalidateTag
 * invalidates every locale of that global.
 */
export const getCachedGlobal = <T extends Global>(slug: T, depth = 0, locale: TypedLocale) =>
  unstable_cache(async () => getGlobal<T>(slug, depth, locale), [slug, locale], {
    tags: [`global_${slug}`],
  })
```

```ts
import { revalidatePath, revalidateTag } from 'next/cache'

import type { Post } from '../../../payload-types'

import { routing } from '@/i18n/routing'

const revalidateLocalizedPath = (path: string) => (locale: string) => {
  revalidatePath(`/${locale}${path}`)
}

const revalidatePathForAllLocales = (path: string) => {
  routing.locales.forEach(revalidateLocalizedPath(path))
}

const revalidateBlogListing = () => {
  revalidatePathForAllLocales('/blog')
  revalidatePath('/[locale]/blog/page/[pageNumber]', 'page')
  revalidatePath('/[locale]/blog/tag/[slug]', 'page')
}

export const revalidatePost: CollectionAfterChangeHook<Post> = ({
  doc,
  previousDoc,
  req: { payload, context },
}) => {
  if (!context.disableRevalidate) {
    if (doc._status === 'published') {
      const path = `/blog/${doc.slug}`

      payload.logger.info(`Revalidating post at path: ${path}`)

      revalidatePathForAllLocales(path)
      revalidateBlogListing()
      revalidateTag('posts-sitemap', 'max')
    }

    // If the post was previously published, we need to revalidate the old path
    if (previousDoc._status === 'published' && doc._status !== 'published') {
      const oldPath = `/blog/${previousDoc.slug}`

      payload.logger.info(`Revalidating old post at path: ${oldPath}`)

      revalidatePathForAllLocales(oldPath)
      revalidateBlogListing()
      revalidateTag('posts-sitemap', 'max')
    }
  }
  return doc
}

export const revalidateDelete: CollectionAfterDeleteHook<Post> = ({ doc, req: { context } }) => {
  if (!context.disableRevalidate) {
    revalidatePathForAllLocales(`/blog/${doc?.slug}`)
    revalidateBlogListing()
    revalidateTag('posts-sitemap', 'max')
  }

  return doc
}
```
